- **Search Module**: Implements a controller-driven architecture (`SearchMediaController`) using GetX for state management. It supports both direct media search and title-based search, integrating SSE (Server-Sent Events) for real-time progress tracking via `SseClient`. Search history is persisted locally using Drift ORM through `SearchHistoryRepository`.
- **Search Result Module**: Provides a reusable result handling layer (`SearchResultController`) that manages filtering, sorting, and view modes for torrent/media items. It relies on Freezed models (`SearchResultItem`) for immutable data structures.
- **Media Detail Module**: Acts as the central hub for media metadata. `MediaDetailController` orchestrates data fetching from multiple sources (TMDB, Douban, Bangumi) and manages local caching via Drift (`MediaDetailCaches`). It integrates with `MediaDetailService` to check library existence, fetch season details, and manage subscription states via `SubscribeService`.
- **Dependency Direction**: Controllers depend on specialized Services and Repositories. Shared models use Freezed for serialization. Network requests are abstracted via `ApiClient`.
